# project-organisation-for-bioinformatics-tutorial

Welcome! If you are stepping into bioinformatics for the first time or just need a refresher, this tutorial is for you.

Bioinformatics work lives on the command line, in shared folders, and in code that has to make sense months later. Before you jump into pipelines and analysis, it helps to get comfortable with a few fundamentals: how to use the terminal, how Linux permissions work, how to keep a project tidy, and how to track your work with Git and GitHub.

This short guide walks you through those basics, with enough hands-on practice.

---

## What you will learn

By the end of this tutorial, you should be able to:

- Navigate and work in the Linux terminal with everyday commands
- Read and change file permissions safely on a shared system
- Organise a bioinformatics project so raw data, code, and results stay easy to find
- Use Git and GitHub to save, share, and collaborate on your work
- Work with branches and pull requests so you and your collaborators do not step on each other’s toes
